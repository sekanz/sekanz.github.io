Goal:  
Build a complete, functional vanilla JavaScript budget app using LocalStorage. The app should track real bank accounts, virtual goal accounts, predictable income streams, paycheck allocation rules, monthly payments, retirement accounts, manual transactions, and include a projection engine. The UI should be minimal; I will handle styling later.
No external libraries should be used.  
All accounts must be editable.
Keep all created files and folders inside the Budget02 folder.

🧱 CORE SYSTEM COMPONENTS
1. Real Accounts (Actual Bank Accounts)
Data Model
Code
{
  id: string,
  name: string,
  balance: number
}
Behavior
Receive income

Fund virtual accounts

Pay monthly payments

Support manual deposits/withdrawals

Support editing existing accounts (name + starting balance)

2. Virtual Accounts (Goal Buckets)
Data Model
Code
{
  id: string,
  name: string,
  balance: number,
  allocationPerPaycheck: number
}
Behavior
Receive automatic allocations from each paycheck

Support manual withdrawals

Do NOT interact with monthly payments

Do NOT affect real account balances

Support editing existing virtual accounts (name + allocation amount)

3. Income Sources (Predictable Income)
Data Model
Code
{
  id: string,
  name: string,
  amount: number,
  frequency: "weekly" | "biweekly" | "monthly",
  nextPayDate: string,
  depositToAccountId: string
}
Behavior
On due date:

Add income to a real account

Apply virtual account allocations

Apply retirement contributions

Update next pay date

Support editing existing income sources (all fields)

4. Allocation Rules
Each virtual account has a fixed amount allocated per paycheck.

Example:

Code
Car Fund: $100 per paycheck
Furniture Fund: $50 per paycheck
Savings Fund: $200 per paycheck
These are stored inside each virtual account.

5. Monthly Payments (Scheduled Expenses)
Data Model
Code
{
  id: string,
  name: string,
  amount: number,
  frequency: "monthly",
  nextDueDate: string,
  payFromRealAccountId: string
}
Behavior
On or after due date:

Deduct from real account

Create a realWithdrawal transaction

Update nextDueDate to next month

Support editing existing monthly payments

6. Retirement Accounts (External Investment Accounts)
Data Model
Code
{
  id: string,
  name: string,
  balance: number,
  contributionPerPaycheck: number,
  employerMatchPercent?: number,
  growthRateAnnual?: number
}
Behavior
On each paycheck:

Add contribution

Add employer match (if defined)

Optionally apply growth

Does NOT reduce real account balances

Does NOT interact with virtual accounts

Does NOT interact with monthly payments

Support editing existing retirement accounts

7. Transactions
Data Model
Code
{
  id: string,
  type: "income" | "realDeposit" | "realWithdrawal" | "virtualWithdrawal",
  amount: number,
  realAccountId: string,
  virtualAccountId?: string,
  date: string,
  note: string
}
🔮 PROJECTION ENGINE (Future Simulation System)
Must be implemented with pure vanilla JS. No external libraries.

Projection Data Model
Projection Event
Code
{
  date: string,
  type: "income" | "monthlyPayment" | "retirementGrowth",
  sourceId: string
}
Projection Snapshot
Code
{
  date: string,
  realAccounts: { [id]: number },
  virtualAccounts: { [id]: number },
  retirementAccounts: { [id]: number },
  totals: {
    real: number,
    virtual: number,
    retirement: number,
    netWorth: number
  }
}
Projection Result
Code
{
  startDate: string,
  endDate: string,
  snapshots: ProjectionSnapshot[]
}
⭐ Projection Algorithm (Vanilla JS)
1. Clone all financial data
Deep copy:

Real accounts

Virtual accounts

Retirement accounts

Income sources

Monthly payments

2. Build a timeline of future events
For the chosen projection window (3, 6, 12 months, or custom):

Generate all future paycheck dates

Generate all future monthly payment dates

Generate retirement growth events

3. Sort events chronologically
4. Process each event
Depending on type:

A. Income Event
Add income to real account

Apply virtual allocations

Apply retirement contributions

Apply employer match

Update next pay date

B. Monthly Payment Event
Deduct from real account

Create a simulated withdrawal

Update next due date

C. Retirement Growth Event
Apply growth rate

5. After each event, record a snapshot
6. Continue until end date
