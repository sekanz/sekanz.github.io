## 💾 Data Persistence

All data stored in browser LocalStorage:

```javascript
localStorage.getItem('budgetApp_realAccounts')      // Array of real accounts
localStorage.getItem('budgetApp_virtualAccounts')   // Array of goals
localStorage.getItem('budgetApp_incomeSources')     // Array of income sources
localStorage.getItem('budgetApp_transactions')      // Array of transactions
```

Persists across:
- ✅ Browser refreshes
- ✅ Closing and reopening browser
- ✅ Multiple browser tabs (same browser)

Does NOT persist across:
- ❌ Different browsers (Chrome vs Firefox)
- ❌ Private/Incognito mode (clears on close)
- ❌ Clearing browser storage/cache

## 🎯 Core Algorithms

### Automatic Paycheck Processing

```
User opens app on Pay Date
  ↓
processDueIncomes() runs
  ↓
For each income: if nextPayDate <= today
  ↓
processIncome():
  1. Add amount to real account
  2. Allocate to virtual accounts:
     - If custom rules: use them
     - Else: add allocationPerPaycheck to each account
  3. Calculate next pay date
     - Weekly: add 7 days
     - Biweekly: add 14 days
     - Monthly: add 1 month
  4. Create transaction record
  5. Save to LocalStorage
```

### Unallocated Money Calculation

```
Unallocated = Total Real Money - Total Virtual Money

Example:
  Checking: $8,000
  Savings: $2,000
  Total Real: $10,000
  
  Car Fund: $2,000
  Furniture: $500
  Emergency: $1,200
  Total Virtual: $3,700
  
  Unallocated: $10,000 - $3,700 = $6,300
```

## 🔐 Security Notes

**What's Safe:**
- ✅ All data stays on your device
- ✅ No internet required
- ✅ No server to hack
- ✅ No passwords transmitted

**What to Consider:**
- ⚠️ Anyone with device access can see data
- ⚠️ Clearing cache deletes all data
- ⚠️ Device-only (not synced to cloud)
- ⚠️ Browser private mode clears on close

**Recommendations:**
- Use browser password manager for backup
- Export data periodically (future feature)
- Don't share device with others
- Consider private mode for privacy
